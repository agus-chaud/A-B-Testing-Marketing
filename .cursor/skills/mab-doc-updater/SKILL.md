# mab-doc-updater

## Cuando usar

- Cambio en instalacion, ejecucion o features visibles.
- Nuevas decisiones tecnicas que deban quedar explicitas.
- Cambios en metricas, metodos estadisticos o interpretacion de resultados.

## Objetivo

Mantener `README.md` y docs alineados al estado real del proyecto.

## Checklist

1. Actualizar secciones de ejecucion/uso si cambio flujo.
2. Reflejar nuevas capacidades o limitaciones.
3. Evitar texto redundante o desactualizado.
4. Mantener mensajes claros y breves.
